# فانوس (Fanous)

> عالمٌ حيّ ينبض داخل فانوس سحري — ينمو معك، ويتغيّر باستمرار، ولا ينتهي أبدًا.

لعبة أندرويد أصلية (APK)، مبنية بالكامل داخل هذا المستودع، تُصدَّر تلقائيًا
عبر GitHub Actions بلا أي خطوة يدوية. الفكرة الكاملة، ولماذا هي مختلفة عن أي
لعبة موجودة حاليًا على متاجر الهواتف، موثّقة في
[`docs/GAME_DESIGN.md`](docs/GAME_DESIGN.md). القرارات التقنية وأسبابها
موثّقة في [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

هذا الملف يغطي الجانب العملي فقط: ما هو موجود، وكيف يعمل، وكيف تشغّله.

## حالة المشروع

هذه هي **المرحلة الأولى (Phase 1)**: البنية التحتية والهيكل المعماري.
المشروع في هذه اللحظة يبني وينشر تطبيقًا حقيقيًا وقابلًا للتثبيت (شاشة
إقلاع متحركة باسم اللعبة)، مع كل الأنظمة الأساسية التي سيُبنى عليها كل شيء
لاحقًا. لا يوجد بعد عالم لعب فعلي — هذا يأتي في المراحل التالية الموضحة في
خارطة الطريق أسفل هذا الملف.

كل ملف مذكور هنا **يعمل فعليًا** وتم التحقق منه محليًا عبر تشغيل حقيقي
لمحرك Godot 4.7.1 (استيراد المشروع كاملًا، وتشغيل خط تصدير Android حتى آخر
خطوة ممكنة دون SDK فعلي) — وليس كودًا مكتوبًا نظريًا فقط.

## كيف يعمل البناء التلقائي

كل push إلى فرع `main` يُشغّل `.github/workflows/build-android.yml`، الذي:

1. يجهّز JDK 17 وAndroid SDK على عامل Ubuntu نظيف.
2. يُنزّل محرر Godot 4.7.1 headless وقوالب التصدير المطابقة (ومخزَّنة مؤقتًا
   Cache للتشغيلات اللاحقة كي لا يُعاد تنزيلها كل مرة).
3. يستورد كل موارد المشروع مرة واحدة (خطوة ضرورية لتفادي مشكلة معروفة في
   Godot عند التصدير من مستودع نظيف — مشروحة داخل ملف الـworkflow نفسه).
4. يُثبّت قالب بناء Android عبر Gradle، ثم يُصدّر نسخة APK للتصحيح (Debug).
5. يتحقق فعليًا من وجود ملف الـAPK الناتج (وليس فقط من رمز خروج الأمر).
6. يرفع الـAPK كمُخرَج (Artifact) قابل للتنزيل من تبويب Actions في GitHub،
   لمدة 30 يومًا.

لا حاجة لأي Secrets أو إعداد يدوي في هذه المرحلة — نسخة Debug تُستخدم مفتاح
تصحيح Godot الافتراضي المدمج تلقائيًا. توقيع نسخة Release الحقيقية لمتجر
Google Play سيُضاف في مرحلة لاحقة عبر GitHub Secrets، بدون تعديل جوهري في
الملف الحالي.

### تنزيل وتثبيت آخر بناء

من تبويب **Actions** في المستودع → افتح آخر تشغيل ناجح لـ *Build Android
APK* → مُخرَج `fanous-debug-apk` أسفل الصفحة → نزّل الملف المضغوط، واستخرج
`fanous-debug.apk`، وثبّته على جهاز أندرويد (يتطلب تفعيل "تثبيت من مصادر
غير معروفة" لنسخ التصحيح غير الموقّعة بمفتاح متجر رسمي).

## التشغيل محليًا (اختياري)

لست مضطرًا لفتح Godot محليًا لأي شيء — كل شيء يعمل عبر GitHub. لكن إن أردت
المعاينة والتعديل المباشر:

1. نزّل [Godot 4.7.1](https://godotengine.org/download) (النسخة القياسية،
   ليست Mono/.NET — المشروع GDScript خالص).
2. افتح `project.godot` من داخل Godot مباشرة.
3. F5 للتشغيل على سطح المكتب لمعاينة سريعة، أو Project → Export لتصدير
   محلي على أندرويد (يتطلب إعداد Android SDK محليًا حسب
   [توثيق Godot الرسمي](https://docs.godotengine.org/en/stable/tutorials/export/exporting_for_android.html)).
